# Personal Hub — Redesign Handoff

Этот пакет содержит всё необходимое для передачи редизайна в разработку (Claude Code или любой другой агент/разработчик).

**Дизайн-направление:** Brutalist / Terminal — плотная mono-типографика, acid-lime акцент на тёмной теме, editorial-структура «газетного разворота» на главной.

---

## 📂 Что внутри

| Файл | Что это |
|---|---|
| `README.md` | Этот файл — навигация |
| `PROMPT.md` | ⭐ **Готовый промт для Claude Code** — скопировать и вставить |
| `HANDOFF.md` | Полная спецификация: скоуп, план миграции в 5 этапов, чеклист PR |
| `TOKENS.md` | Дизайн-токены: цвета, шрифты, spacing, стили кнопок/тегов |
| `DATA-MAP.md` | Откуда берутся данные для каждого блока Today + алгоритм Priority_01 |
| `mockups/hub-brutalist-v2.html` | ⭐ **Финальный мокап** — открывать в браузере для сверки |
| `mockups/hub-brutalist.html` | Референсный мокап с Vitals (для будущего, когда подключится Garmin) |

---

## 🚀 Как использовать

1. **Скопируйте папку `handoff/` в корень репозитория `my-personal-hub`**
2. Откройте `mockups/hub-brutalist-v2.html` в браузере — убедитесь, что это та визуальная цель, которую вы хотите
3. Откройте `PROMPT.md` → скопируйте промт → запустите Claude Code в репозитории → вставьте
4. Claude Code сделает Этап 1 (токены + shell). **Ревью → мердж → следующий этап.**

---

## 🗂 Порядок чтения для Claude Code

1. `PROMPT.md` (этот файл — стартовая инструкция)
2. `HANDOFF.md` (скоуп, этапы, чеклист)
3. `TOKENS.md` (копировать в `tokens.css`)
4. `DATA-MAP.md` (при работе над Today)
5. `mockups/hub-brutalist-v2.html` (визуальный эталон)

---

## ⚠️ Важно

- **Финальный мокап — `hub-brutalist-v2.html`**, не v1. В v2 Vitals скрыт (Garmin не подключён).
- **Vitals НЕ удаляется из кода** — только скрывается в UI. В `DATA-MAP.md` есть раздел «Когда Vitals вернётся».
- **Данные в мокапе — демо-портфолио**. Реальные цифры должны приходить из БД.
- **Дизайн-токены и структуру Today не менять** без согласования.
